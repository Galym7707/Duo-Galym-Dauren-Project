# Project Brief

## Где мы сейчас

26 марта 2026. До подачи осталось 8 дней, до финала 14 дней.

## Ближайшая цель

Собрать submission-level MVP, который уверенно проходит через demo loop:

`загрузка данных -> нормализация -> аномалия -> incident card -> задача -> MRV-отчет`

## Главный риск

Потратить время на "enterprise-красоту" вместо понятного workflow, который жюри увидит за 90 секунд.

## North Star

Мы строим не карту и не chatbot. Мы строим MRV-инструмент для methane & flaring visibility, который помогает:

- быстро обнаружить аномалию
- завести инцидент
- запустить проверку
- подготовить отчетность

## CTO-фильтр

Любая фича проходит через четыре вопроса:

1. Реально ли сделать это качественно к 3 апреля?
2. Не развалится ли это на демо?
3. Есть ли простой fallback?
4. Усиливает ли это главный demo loop?

## Pitch-фильтр

Любая фича проходит через четыре вопроса:

1. Поймет ли это топ-менеджер нефтегаза за 30-60 секунд?
2. Выглядит ли это как реальный инструмент, а не студенческий дашборд?
3. Дает ли это wow-эффект без переобещания?
4. Укрепляет ли это доверие к продуктовой зрелости?

## Что говорить

- MRV tool for methane and flaring visibility
- Screening and operational prioritization layer
- Pre-LDAR intelligence layer
- ESG / compliance / operations bridge

## Что не говорить

- Полная замена LDAR
- Точное определение конкретного вентиля
- Магический AI
- Еще один dashboard

## Реалистичный scope

Ограничения конвертируются в силу позиционирования:

- Sentinel-5P не показывает конкретный вентиль
- Поэтому мы честно позиционируемся как screening layer
- Это снижает риск пилота и ускоряет внедрение
- Это звучит зрелее для жюри, чем завышенные обещания

## Приоритеты

### Сейчас

- Карта и слои CH4 / flare
- Индекс аномалии
- Event / incident card
- Создание задачи
- Экспорт MRV-отчета
- Понятный narrative для 60-90 секунд скринкаста

### Потом

- Полировка UI
- Более сильная экономика и ROI framing
- Глубже Q&A для регулятора / инвестора / ESG-аудитора

### Не трогать для MVP

- Сложный auth
- RBAC
- Mobile app
- Микросервисы
- Kubernetes
- Realtime streaming
- Тяжелый ML
- Enterprise-интеграции, которые невозможно показать на сцене

## Конкурсная формула

Наша дифференциация:

`аномалия -> инцидент -> задача -> MRV-отчет`

Если нужно выбирать между красивой аналитикой и понятным workflow, выбираем workflow.
